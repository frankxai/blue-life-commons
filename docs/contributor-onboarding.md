# Contributor Onboarding

Welcome. You're not joining a community — you're producing an artifact that becomes public knowledge, partner leverage, or funded impact.

## Your first 30 minutes

1. **Read the public promise** in the [README](../README.md).
2. **Pick an artifact class** from [CONTRIBUTING.md](../CONTRIBUTING.md) — species page, region briefing, field mission, research summary, translation, lesson…
3. **Find an issue** labeled `good-first-issue` or `agent-ready`, or open one with an issue template.
4. **Read the three standards:** [ETHICS.md](../ETHICS.md), [SOURCES.md](../SOURCES.md), [STYLE.md](../STYLE.md).
5. **Copy a template** (e.g., [missions/templates/field-mission-template.md](../missions/templates/field-mission-template.md)) and create your artifact with complete metadata frontmatter.
6. **Open a PR** and fill in the checklist.

## Using a coding agent

This project is agent-native. Open Claude Code, Codex, or Cursor and run something like:

```
Read AGENTS.md and issue #42.
Create the requested region briefing.
Use existing templates.
Add sources.
Do not make unsupported claims.
Open a PR.
```

The agent rules live in [AGENTS.md](../AGENTS.md); role briefs live in [/agent](../agent/).

## What happens after you submit

```
PR → Review (sources, ethics, clarity) → Merge → Published to website
   → Impact record + contributor credit → Eligible for sponsorship/grants
```

Your GitHub handle is recorded in the artifact's `contributors` metadata and surfaces on your contributor profile and the impact ledger.

## Where to ask questions

Live discussion happens in community channels (Discord), but decisions and truth live in GitHub. If your question reveals a gap — missing docs, unclear template — open an issue. That's an artifact too.
